# Ainekio Documentation Authority

The normative set is:

- `Ainekio - System Specification v1.0.docx` - behavior, protocol, software,
  safety, and acceptance authority.
- [Parts Overview](https://docs.google.com/document/d/1wz0kyqttPK3HHL0P0_9lLtRW_B87U5u4kEt-UhttHFA/edit?usp=sharing) - externally maintained electrical-facts authority named by the specification.

The Parts Overview is available externally and was reviewed for the 2026-07-15
firmware-preparation pass. Its planned parts and electrical values may be used
to prepare configuration and test procedures. They are not installed-hardware
evidence: delivered markings, wiring, measurements, photos, and H-series results
must still confirm them before the initial reduced-range profile is expanded or
accepted as physically validated.

`SLAVE_BRAIN_PROGRESS.md` records implementation status and evidence but is not
normative. `REPOSITORY_MAP.md`, older specifications, archived notes, and the
Sesame reference material are background or historical inputs only.

`LOCAL_WAKE_WORD.md` is the current owner-local microWakeWord training,
packaging, installation, and validation procedure. It does not override the
normative system specification or hardware acceptance gates.
